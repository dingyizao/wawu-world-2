# 娃屋世界 2 Demo 设计规格

状态：已完成交互式评审，等待书面规格确认

日期：2026-06-30

依据：`娃屋世界_PRD_记忆碎片版_V3.0.md`

## 1. 目标与产品边界

“娃屋世界 2”是一个移动优先的 Web App Demo。它用高德真实地图承载现实步行，并让玩家与具有独立 MBTI 人格的 AI 分身作为两个角色共同探索城市。

核心循环：

> 共同走路 → 领取记忆碎片 → 分身观察并提出地点建议 → 玩家确认 → 打卡探秘 → 获得物品与故事 → 收进娃屋储物柜 → 查看共同散步回顾

首版采用“全量功能展示 + 核心闭环真实可玩”：

- 核心闭环必须可操作、可持久化、可验证。
- PRD 的 P0/P1 入口均出现；非核心模块允许使用浅层交互或受控模拟。
- 记忆碎片是唯一核心游戏资源。创世点数不进入本 Demo 的可交易流程。
- 不做复杂自由建造、公开精确轨迹、强排行榜或旧版多货币经济。

## 2. 首次入世与 MBTI 分身

所有新访客必须完成入世流程，不能一键跳过：

1. 遇见分身：选择推荐形象并起名。
2. 选择人格：直接选择 16 种 MBTI 之一；不了解 MBTI 时可选“旅途中形成”。
3. 查看 Persona Tag：系统把 MBTI 映射为 3–4 个可理解的行为标签。
4. MBTI+ 微调：调整主动程度、表达密度和自主行动倾向。
5. 选择关系：另一个我、同行伙伴、记录搭档或守护伙伴；默认同行伙伴。
6. 说明收益后请求定位与步数权限。
7. 完成第一次 100 步真实或明确标注的训练同行。
8. 领取第一批记忆碎片，花 1 片触发分身第一次观察，解锁完整世界。

每个 MBTI 使用独立人物头像和地图同行形象。头像表达人格，但外观与人格可拆分：玩家可“保留性格、换外观”。MBTI 影响分身语气、主动程度、地点偏好、行动选择和日报叙述，不只作为标签展示。

## 3. 地图共同探索

地图为默认首页。高德 JS API 2.0 提供真实底图；玩家位置使用定位光点，AI 分身使用在光点附近动画跟随的人物，不伪造第二个 GPS 坐标。

开始同行后：

- 客户端采集位置、步数和时段。
- 服务端模糊起终点并查询附近 POI、路线与已解锁城市格。
- Persona 策略结合 MBTI、天气、时段、路线和历史记忆选择值得观察的内容。
- Coze 原生模型生成一句短观察或建议；失败时使用同人格的审核模板并标记来源。
- 分身可建议绕路、打卡、拍照或稍后探索，但玩家决定是否执行。
- 结束同行后生成双人回顾：路线摘要、步数、对话、城市发现、碎片变化和新物品。

成都 Demo 提供一条 10 锚点主题路线。真实定位可用于到场打卡；无定位或非成都评审者可进入明确标注的训练同行，训练结果不冒充真实 LBS 解锁。

## 4. 页面与功能结构

底部导航保持四项：

- 城市地图：真实底图、城市格、附近锚点、同行会话、路线、打卡、周挑战和城市明信片。
- 我的分身：MBTI 状态、目标、记忆、关系、五类行为、离线日报、待确认事件和记忆兑换/创作台。
- 我的娃屋：作为储物柜，保存家具、服装、纪念品、明信片和故事物件；支持筛选、预览、查看来源、装备、展示和回收，不做复杂房间建造。
- 我的：记忆碎片流水、城市图鉴、隐私与定位设置、账号恢复和数据导出。

创作台位于“我的分身”。它使用记忆碎片调用 Coze 生成描述或概念结果，生成/兑换出的可见物品进入娃屋储物柜。

## 5. 视觉与 Imagegen 资产

视觉锚点来自参考项目：

- 细密线稿、宣纸质感、青绿山水和成都街巷。
- 现代国风动画人物，温和表情、清晰轮廓、真实布料与随身物件。
- 旧物单体采用木纹、藤编、铜件、朱砂和湖水青点色。
- 品牌色延续宣纸白、朱砂红、湖水青、记忆金和墨褐。

高德底图不套重滤镜；道路、地名和缩放保持可读。品牌感来自 Imagegen 生成的分身、POI 标记、记忆碎片、物品、徽章、场景和装饰。

首版资产清单：

- 32 个 MBTI 角色资产：16 个选择页头像 + 16 个地图同行形象。
- 24 个地图资产：记忆碎片状态、10 类 POI、打卡章、路线章、锁定/发现/探秘标记。
- 30 个储物柜物品：家具、服装、纪念品、明信片和成都文化物件。
- 10 个场景/状态插画：入世、共同散步回顾、日报、空储物柜、地图与 AI 降级等。

每个资产单独生成、单独命名、单独验收，不使用整张 AI UI 截图。图片中不生成动态文字。透明素材优先采用内置 Imagegen 的纯色背景加本地去背流程。

## 6. 技术架构与公开接口

技术栈沿用参考项目已验证的 Next.js App Router、TypeScript、Vitest、Playwright 和 Coze Web 部署结构，但新项目不复制旧版业务页面。

主要服务模块：

- 账户与入世门禁
- 步数与记忆碎片原子账本
- 同行会话协调器
- 城市格与打卡校验
- MBTI Persona 行为策略
- 储物柜与兑换
- 高德与 Coze 适配器
- Coze DB/对象存储仓库

主要 API：

- `GET /api/state`
- `POST /api/onboarding/complete`
- `POST /api/walks/start`
- `POST /api/walks/observe`
- `POST /api/walks/finish`
- `POST /api/anchors/check-in`
- `POST /api/agent/actions`
- `POST /api/creation`
- `GET /api/inventory`
- `POST /api/inventory/:id/equip`
- `GET /api/health`

核心数据对象：

- `UserProfile`：教程阶段、隐私设置、碎片余额。
- `AgentProfile`：头像、MBTI、Persona Tag、关系和行为微调。
- `WalkSession`：真实/训练模式、模糊路线、步数、共同发现和对话。
- `CityAnchor`：高德 POI、城市格和锁定/发现/探秘状态。
- `ShardLedgerEntry`：获得/消耗、原因、关联对象和幂等键。
- `InventoryItem`：素材、类别、来源、稀有度、装备和展示状态。

记忆碎片扣减、行为创建和物品发放必须在同一事务或等价原子操作中完成，避免重复领取和负余额。

## 7. 高德、Coze 与密钥配置

- 高德 JS API Key 与安全配置仅通过本地/部署环境注入，并在高德控制台限制合法域名。
- 高德 Web 服务 Key 只在服务端使用，负责 POI、地理编码和步行路线。
- 高德 MCP 按官方推荐使用 Streamable HTTP 配置，服务于开发期地点核对和路线设计；线上 App 不把 MCP 当作唯一运行依赖。
- Coze 原生模型用于地点观察、分身对话、日报和创作台；每项能力都有确定性的预置降级。
- 任何凭证、精确路线、私人照片或录音都不得进入 Git 历史。

当前 Codex 没有可直接安装的高德插件；实施阶段将按官方 MCP 教程添加窄范围本地配置，而不是安装不相关插件。

## 8. 失败状态与隐私

- 地图加载失败：显示 Imagegen 成都探索图和训练锚点，标记“地图降级模式”。
- 定位/步数不可用：保留只读地图，允许训练同行，不生成真实打卡记录。
- Coze 失败：使用按 MBTI 审核的模板，标记“预置内容”。
- 生产存储不可用：健康检查失败，不能显示“云端保存成功”。
- 余额不足：行为保持未执行，账本无扣款。
- 重复请求：通过幂等键返回原结果，不重复发放物品。

精确路线默认私密，保存前模糊起终点。分身只能提出建议；公开、付费、隐私和路线改变必须由玩家确认。

## 9. 测试与验收

测试先行覆盖：

- 单元测试：碎片兑换、余额不足、重复请求、MBTI 行为策略、物品状态和打卡距离。
- API 集成：入世门禁、同行开始/结束、打卡、账本、储物柜、Coze 降级和错误码。
- 浏览器旅程：创建分身 → 训练/真实同行 → 领取碎片 → 地点提议 → 玩家确认 → 打卡 → 获得物品 → 储物柜 → 回顾。
- 视觉验收：移动端主要页面、16 个 MBTI 卡片、地图覆盖物、储物柜和空/错误状态。
- 部署冒烟：生产构建、健康接口、高德真实底图、Coze 真实/降级、无控制台错误和无密钥泄漏。

完成实现后开启两轮对抗式审查：

1. 产品审查：核心循环、PRD 一致性、单一资源、MBTI 行为真实性、共同走路沉浸感和浅层入口诚实度。
2. 工程审查：隐私、密钥、账本原子性、重复请求、移动端可访问性、模型/地图降级和部署健康。

高优先级问题必须修复并重新验证后才允许发布。

## 10. 发布与暂停条件

本地通过全部检查后创建 GitHub 仓库并推送。Coze 部署复用参考项目的 Web profile、构建/运行脚本和健康检查经验，但使用独立项目标识。

遇到验证码、2FA、扫码、生产应用选择、域名白名单、商业许可、付费额度、不可逆发布或额外敏感数据上传时暂停，等待玩家明确操作。
