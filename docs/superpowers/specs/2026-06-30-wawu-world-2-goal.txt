/goal 严格依据《娃屋世界 PRD：步数·记忆碎片·LBS 城市探秘》V3.0，在当前新项目中交付“娃屋世界 2”移动 Web App Demo：复用参考项目的技术经验和国风绘本视觉语言，以高德真实地图为主入口，实现严格新手门禁、16 种 MBTI AI 分身、玩家与分身双角色共同走路、步数兑换唯一资源记忆碎片、城市格与锚点探秘、分身行为与日报、Imagegen 独立美术资产、创作台、娃屋储物柜、Coze 原生模型及明确降级，并完成对抗式审查、GitHub 上传和 Coze 公开部署。
验证：运行项目提供的 lint、typecheck、unit test、integration test 和 production build；启动本地应用，使用移动端浏览器完整走通“创建分身→训练或真实同行→领取记忆碎片→分身地点提议→玩家确认→打卡探秘→获得物品→储物柜查看→散步回顾”；验证高德真实底图、POI 或步行路线服务、Coze 真实生成与预置降级；检查无控制台错误、无密钥入库、健康接口通过；保存移动端截图、测试输出、GitHub 仓库链接和 Coze 公网链接作为完成证据。
约束：只保留记忆碎片一种核心游戏资源；不恢复旧版多货币、复杂自由建造、精确轨迹公开或强排行榜；动态文字、数值、布局和点击区域由代码渲染，美术头像、地图标记、物品、徽章、场景和装饰由 Imagegen 独立生成；真实定位、公开内容、付费和隐私行为必须由玩家确认；AI 失败必须明确标注降级，不伪装真实生成。
边界：只写入当前“wawu world 2”项目和为该项目明确需要的 Codex 高德 MCP 配置；只读取“wawu world”作为风格、架构与部署经验参考，不修改参考项目；高德和 Coze 密钥仅写入本地或部署环境变量，不提交 GitHub；不改动其他仓库、账号、应用或系统配置。
迭代策略：按测试先行完成一个可验收闭环，再扩展全量展示页；每次有意义改动后重跑最小相关检查；Imagegen 每个资产单独生成、检查并登记，失败只重做目标资产；完成后开启至少两轮对抗式审查，分别检查产品闭环、隐私安全、密钥泄露、移动端交互、降级真实性和部署可用性；同一故障最多做三轮有新证据的修复。
完成条件：全部核心页面和浅层展示入口可访问，核心闭环可真实操作；16 种 MBTI 分身与双角色同行成立；高德、Coze、账本、储物柜和降级路径均有运行证据；相关检查和生产构建通过；对抗式审查中的高优先级问题已修复；代码已上传 GitHub，Coze 公网地址可打开并通过健康检查。
暂停条件：需要新的付费额度、验证码、2FA、扫码、账号授权、生产应用选择、域名白名单、商业许可确认、敏感数据上传、不可逆发布或密钥缺失时暂停并向用户说明具体动作；若同一外部阻塞在三轮有证据尝试后仍无法解决，保留可运行本地成果并报告阻塞。

默认选择理由：采用“全量功能展示 + 核心闭环真实可玩”的保守方案，既覆盖 PRD，又不以伪造集成换取页面数量。

/goal Deliver a mobile-first Wawu World 2 web app demo in the current project, strictly based on the V3.0 PRD and informed by the reference project's proven visual, architectural, and deployment patterns. The demo must use an AMap-powered map as the main surface, enforce onboarding, support 16 MBTI AI companions, create an immersive dual-character walking experience, keep memory shards as the only core resource, cover city exploration, companion actions and reports, an Imagegen asset library, a creation desk, a storage-cabinet dollhouse, real Coze generation with explicit fallbacks, adversarial review, GitHub publication, and Coze deployment.
Verification: Run the project lint, typecheck, unit tests, integration tests, and production build; start the app and complete the mobile journey from companion creation through walking, shard claiming, companion POI suggestion, player confirmation, check-in, item reward, storage viewing, and walk recap; verify AMap rendering and services, Coze generation and deterministic fallback, console cleanliness, secret exclusion, and the health endpoint; capture screenshots, command output, the GitHub repository URL, and the public Coze URL.
Constraints: Keep memory shards as the only core gameplay resource; do not reintroduce the legacy multi-currency economy, complex free-form building, public precise routes, or competitive rankings; render dynamic text, values, layout, and hit targets in code while generating each visible art asset independently with Imagegen; require player confirmation for location-sensitive, public, paid, or private actions; label fallbacks honestly.
Boundaries: Write only inside the current Wawu World 2 project and the narrowly required Codex AMap MCP configuration; read but never modify the Wawu World reference project; store AMap and Coze credentials only in local or deployment environment variables and never in Git; do not change unrelated repositories, accounts, applications, or machine settings.
Iteration policy: Build one test-proven core loop first, then add the complete set of showcase pages; rerun the smallest relevant checks after meaningful changes; generate and validate Imagegen assets one by one and retry only the failed asset; run at least two adversarial review rounds covering product coherence, privacy, secret leakage, mobile interaction, honest fallback behavior, and deployment; make at most three evidence-driven repair attempts per repeated failure.
Stop when: The core loop is genuinely playable, all required showcase surfaces are reachable, the 16 MBTI companions and dual-character walking experience are demonstrated, AMap, Coze, ledger, storage, and fallback paths have runtime evidence, checks and production build pass, high-priority adversarial findings are fixed, GitHub contains the code, and the public Coze URL passes its health check.
Pause if: New paid quota, OTP, 2FA, QR confirmation, account authorization, production application selection, domain allowlisting, commercial licensing, sensitive-data upload, irreversible publication, or missing credentials require human action; after three evidence-driven attempts at the same external blocker, preserve the working local result and report the blocker.
