# 娃屋世界 2：产品对抗式审查

审查日期：2026-07-03

## 结论

核心闭环已可演示：严格入世 → MBTI 分身 → 前台真实/训练同行 → 步数增长 → 位置重合自动拾取 → 分身建议 → 玩家确认 → 打卡 → 虚拟物品入柜 → 同行回顾。全量页面用于展示产品结构，非核心模块按 PRD 约定保持浅交互。

## 保留限制

- 路线、挑战、明信片、账户恢复和数据导出属于全量功能展示，当前仅提供浅层演示。
- Web Demo 只统计当前同行页面前台运行期间的运动/GPS 步数；系统健康累计步数、后台定位和页面关闭后的持续运行需要原生 App。
- GPS 估算会受室内信号和手机硬件影响；界面与回顾始终显示来源，不把估算伪装成系统步数。
- 地图不会自动弹定位权限；用户需要点击“刷新定位”或开始真实同行后才会请求浏览器定位。
- 浏览器位置可被开发者工具伪造；本 Demo 用服务端奖励、定位精度、样本时效、25 米双确认和幂等降低普通篡改风险，设备级证明留给原生版本。
- 真实到场打卡规则已实现，但成都以外评审应使用训练模式。
